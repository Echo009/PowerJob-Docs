## 整体架构

PowerJob 采用 Server-Worker 架构设计：

```
┌─────────────────────────────────────────────────────────┐
│                    PowerJob Server                       │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐      │
│  │   调度器     │  │   工作流    │  │   控制台    │      │
│  │  Scheduler  │  │  Workflow   │  │   Console   │      │
│  └─────────────┘  └─────────────┘  └─────────────┘      │
│                         │                                │
│                    ┌────┴────┐                          │
│                    │  数据库  │                          │
│                    └─────────┘                          │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │  Worker  │    │  Worker  │    │  Worker  │
    │  应用 A   │    │  应用 B   │    │  应用 C   │
    └──────────┘    └──────────┘    └──────────┘
```

## 核心组件

### Server（调度服务器）

- **调度器**：负责任务的定时调度和触发
- **工作流引擎**：处理 DAG 工作流的编排和执行
- **控制台**：提供 Web UI 管理界面

### Worker（执行器）

- **任务执行**：接收调度请求并执行任务
- **日志上报**：将执行日志上报到 Server
- **心跳保活**：定期向 Server 发送心跳
